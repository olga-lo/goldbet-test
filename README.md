## Основные пользовательские сценарии

| # | Название сценария | Почему важно |
|---|---|---|
| 1 | Регистрация через Email (Happy Path) | Если основной сценарий не работает, то форма не выполняет задачу. Проверяет: email, пароль, валюта, бонус, соглашения, редирект в ЛК |
| 2 | Регистрация через Phone | Альтернативный метод, быстрая верификация. Если не работает, то потеря части аудитории в пользу конкурентов |
| 3 | Регистрация через Socials (Google, Facebook, Telegram) | Популярный метод входа для молодой аудитории. Если не работает, то потеря пользователей |
| 4 | Регистрация с применением промокода | Промокоды это канал привлечения пользователей. Если не работает, пользователь не получит бонус |
| 5 | Регистрация с выбором бонуса | Бонус мотивирует пополнить первый депозит. Ошибка в бонусе это прямые финансовые убытки |

| 7 | Повторная регистрация дубликат Email/телефона | Нельзя создать два аккаунта на один email или телефон. Это защита от мошенничества: один человек не получит два бонуса. Без этой проверки казино потеряет деньги и лицензию |
| 8 | Регистрация без чек-бокса 18+ | Юридическое требование. Гемблинг только для совершеннолетних. Нарушение = штрафы, аннулирование лицензии |

## Баг 3: Несоответствие валидации поля промокода при вводе с клавиатуры и при вставке из буфера обмена

**Шаги воспроизведения:**
1. Открыть форму регистрации https://goldvexy.com/registration

Ввод с клавиатуры и вставка из буфера должны обрабатываться одинаково. Если фильтр по клавиатуре запрещает буквы, вставка тоже должна запрещать. Если вставка разрешена, ввод с клавиатуры тоже должен быть разрешён.

**Серьёзность (Severity):** Medium  
**Приоритет (Priority):** Medium

**Окружение:** Браузер Chrome 125, Windows 11

**Комментарий для разработчиков:** Уточнить требования: промокод должен содержать только цифры или буквы тоже допустимы? После уточнения привести оба способа ввода к одному поведению.

# Самые дорогие баги для бизнеса в гемблинге

## Топ-3 по степени убытков

| Место | Баг | Тип убытков |
|---|---|---|
| 1 | Регистрация несовершеннолетнего | Лицензия, уголовное дело, медиа |
| 2 | SQL-injection / XSS | Утечка данных, GDPR, компрометация |
| 3 | Дубликаты аккаунтов | Прямые финансовые потери |

## Детализация

### 1. Регистрация несовершеннолетнего

**Тип убытков:**
- Штрафы регулятора: –+ за каждое нарушение
- Аннулирование лицензии
- Уголовное дело для директоров
- Медиа-скандал
- Компенсации семьям

**Почему самый дорогой:** Юридические последствия фатальны для всего бизнеса.

---

### 2. SQL-injection / XSS

**Тип убытков:**
- Утечка данных всех пользователей
- Кражa сессий админов
- GDPR-штрафы: до 4% глобального оборота
- Репутационные потери

**Почему дорогой:** Компрометация всей системы.

---

### 3. Дубликаты аккаунтов

**Тип убытков:**
- Один человек получает два приветственных бонуса
- При 1000 регистраций в день и 0.5% мультиаккаунтов = – потерь ежемесячно
- Бонусный абуз масштабируется

**Почему дорогой:** Прямые финансовые потери, постоянные.

---

## Вывод

Баг 1 и 2 могут уничтожить бизнес полностью. Баг 3 вызывает постоянные убытки. Все три нужно тестировать в первую очередь.
